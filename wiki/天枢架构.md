## 结构概览

- **用户意图** → **三层决策中枢**（天枢殿、紫微台、璇玑府）→ **七大执行部**
- **赏罚台**：跨层约束机制，由用户反馈驱动

### 三层决策中枢

| 模块 | 角色 | 功能 |
|------|------|------|
| 天枢殿 | 决策中心 / 大脑 | 理解用户意图，生成任务步骤 |
| 紫微台 | 监督、审核、安全 | 检查步骤是否危险，是否可执行 |
| 璇玑府 | 总指挥、执行分发 | 把任务分给各个功能模块 |

### 七大执行部

| 部门 | 职责 |
|------|------|
| 启元部 | 打开软件、窗口管理 |
| 文翰部 | 输入、文本、搜索 |
| 玄枢部 | 鼠标、点击、拖拽 |
| 金匮部 | 文件、文件夹操作 |
| 监正部 | 错误处理、重试、回滚 |
| 天机部 | 屏幕识别、UI 自动化、坐标 |
| 外交部 | 钉钉、飞书、企微、QQ 等第三方对接 |

### 机制

- **赏罚台**：用户对决策/审核/调度/执行评分奖励，影响各层策略与优先级

如有建议或想法，欢迎通过 Issue 或 PR 参与讨论。
